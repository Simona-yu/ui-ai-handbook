二、可追溯知识库

以后每篇文章、每个观点都会按这个结构记录：

观点 → 方法 → 操作步骤 → 来源链接 → 适用阶段

观点 01：Skill 的本质不是工具，而是设计规则

适用阶段：阶段1 Skill｜状态：已纳入手册 v2.0
```
观点

对 UI 设计师来说，Skill 不是“让 AI 随便帮我设计”，而是把你的设计判断、流程、约束和输出标准写成固定规则。

方法

把高频设计任务拆成固定输入和固定输出，例如后台页面、大屏布局、App 页面方案。

操作步骤
  1. 选择一个高频任务。
  2. 定义输入字段：页面类型、行业、风格、用户、目标。
  3. 定义输出结构：信息架构、模块划分、组件、交互、落地建议。
  4. 保存成固定 Skill 模板。
  5. 每次项目复用并持续优化。

观点 02：你当前不需要先做 Agent

适用阶段：阶段1 → 阶段2｜状态：已纳入手册 v2.0
```
观点

Agent 很有吸引力，但在 Skill 没稳定之前，直接做 Agent 成本高、收益不稳定。

方法

先用 ChatGPT + 固定 Skill 模板跑通流程，再考虑半自动化，最后再进入 Agent。

操作步骤
  1. 先建立 App、大屏、后台三个方向的 Skill。
  2. 用真实项目测试输出是否稳定。
  3. 把好用的输出结构沉淀进手册。
  4. 当每天重复任务明显增多后，再考虑自动化。
来源链接

来自前期关于 Codex、API、Claude Code、Skill、Agent 的讨论总结。

```

观点 03：AI 的价值不是单次出图，而是批量生产设计方案

适用阶段：阶段2 半自动化｜状态：已纳入手册 v2.0
```
观点

对 UI 设计师来说，AI 最大价值不是偶尔生成一张图，而是让方案、结构、风格探索变成批量化流程。

方法

把一个页面扩展成多个方向，把一个项目扩展成多个页面，把一次提案扩展成多套方案。

操作步骤
  1. 输入同一个页面需求。
  2. 要求 AI 生成 3-5 种结构方案。
  3. 要求每个方案说明优缺点。
  4. 选择最适合的方向进入 Figma 精修。
  5. 把可复用结构写入 Skill。

三、Skill 模板库

1. 后台系统页面生成器

【后台页面 Skill】

你是一名资深 B 端 UI/UX 设计师，请为我设计一个后台管理系统页面。

输入信息：

* 页面类型：
* 所属行业：
* 使用角色：
* 核心任务：
* 数据复杂度：低 / 中 / 高
* 风格倾向：简洁 / 科技 / 金融 / 政企

请按以下结构输出：

1. 页面目标
2. 页面信息架构
3. 页面模块划分
4. 表格字段设计
5. 筛选条件设计
6. 主要操作按钮
7. 弹窗 / 抽屉 / 表单建议
8. 空状态 / 加载 / 错误状态
9. 交互注意事项
10. Figma 落地建议

2. 可视化大屏布局生成器

【可视化大屏 Skill】

你是一名可视化大屏设计专家，请帮我规划一个数据可视化大屏。

输入信息：

* 行业：
* 大屏主题：
* 核心数据：
* 使用场景：汇报 / 监控 / 指挥 / 分析
* 风格倾向：科技蓝 / 深色发光 / 政务风 / 极简数据风
* 屏幕比例：16:9 / 超宽屏 / 自定义

请按以下结构输出：

1. 大屏核心叙事
2. 整体布局结构
