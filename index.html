一、三阶段成长路线

当前重点：Skill 下一步：半自动化 未来方向：Agent
核心目标：把你的设计经验，变成 AI 可以反复执行的规则系统。

阶段 1：Skill

建立固定提示词模板，让 AI 按你的设计方法输出。

阶段 2：半自动化

把单次设计变成批量生成，形成稳定工作流。

阶段 3：Agent

让 AI 执行完整设计任务链路，但不建议过早进入。

二、第一阶段：提示词 / Skill

这是你当前最应该投入的阶段。不需要配置环境，不需要 API，也不需要做复杂 Agent。

这一阶段的目标

  • 把常见设计任务拆成固定流程。
  • 沉淀自己的 UI Skill 库。
  • 让 AI 从“随机回答”变成“按规则产出”。

核心原则

  • Skill 必须有固定输入。
  • Skill 必须有固定输出结构。
  • Skill 要解决真实高频任务。
  • 不要写“帮我设计一个 UI”这种泛提示词。

四、第二阶段：半自动化设计

第二阶段的核心是：从“单次生成”变成“批量生成”。

你要建立的能力

  • 批量生成多个页面结构。
  • 同一页面生成多种风格提案。
  • 把常见项目沉淀为模板。
  • 用表格、文档、Prompt 库管理设计输入。

五、第三阶段：Agent / 自动化

第三阶段不是现在的重点。只有当你的 Skill 已经稳定，并且有大量重复任务时，才值得进入。

适合进入 Agent 阶段的条件

  • 每天重复做同类设计任务。
  • 已经有稳定的 Skill 模板。
  • 有明确商业目标或效率目标。
  • 愿意投入时间配置 API、自动化工具或代码环境。
现阶段不要追求全自动，先追求稳定、可复用、能持续积累。

六、你的推荐工作流

  1. 接到项目后，先判断属于 App / 大屏 / 后台哪一类。
  2. 选择对应 Skill 模板。
  3. 填入业务背景、页面类型、风格要求。
  4. 让 AI 输出结构方案。
  5. 你在 Figma 中搭建、筛选、精修。
  6. 把好用的提示词继续沉淀进本手册。

七、避坑清单

  • 不要一开始就做 Agent。
  • 不要把 Skill 写得太泛。
  • 不要每次重新问，要沉淀模板。
  • 不要把 AI 当最终审美决策者。
  • 不要追求工具复杂度，先追求工作流稳定。
